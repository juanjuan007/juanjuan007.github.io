### 一模

9：00～14：00 / 四題

事情是這樣的 : Wiwiho 昨天分享比賽時間分配，她會先看完所有題目，然後再寫題目，分配時間等。

我遵循她的做法試試，畢竟我沒打過五小時的比賽，整體完全不會分配時間。

（題敘是合法的嗎）
基於我沒紀錄submission time，不過大概是可信的。

好了，爆雷開始了

一開場，看了pA，沒有很直覺的想法，想了一下覺得greedy應該可以，但沒有很完整的想法。

接著去看pB，看起來求LCA然後多紀錄一個值就可以的樣子。

再來看pC，又是沒有很好的做法，看看有什麼子題撈撈。

Subtasks 1 : 可以用 dfs 枚舉子集的方法。

Subtasks 2 : 等價於 01背包。

Subtasks 4 : 可以砸樹上dp。

到現在已經過了20幾分鐘，然後我一分都還沒拿到，超慌的，全部人都在狂寫，是我不太適合wiwiho的戰術（？），所以我決定先不要再往下看，先寫完 pC 子題，我覺得比較有把握能拿到分，先寫了subtasks 3，拿了32/100，寫完我發覺應該去看第四題，等到都不會再撈子題。


於是去看pD，

只要按照題敘，寫一個暴力做法，就可以拿到45／100，這真的有點太水了。

我直覺這題我是可以寫出來的，把它想成一個環狀佇列，就可以搭配 BIT 維護，理論可行，開始實踐。

寫完後，發現範測就 WA，奇怪了，是我 0-based,1-based 沒處理好還是什麼，把所有都 print 出來檢查，終於被我抓到我 BIT 刻錯，我不懂為什麼會犯這種奇怪的錯誤，沒記錯的話我寫了這個東西，呆了。

是太緊張還是怎樣ww,我平常沒寫錯過這個。

```cpp
int query(int id){
  int x = 0;
  for(;id;id-=lowbit(id)) x += lowbit(id); // 
  return(x);
}
```

至此，改完這個智障的錯誤，就拿到了 100／100，大約已經過了快一個半小時。

再來，我去寫pB，整體過程是很順利的，沒有寫出什麼bug，一次就AC了。

現在剩下不到三小時，在我手上有 pA，pC。

去想pA，要怎麼寫，我很直覺的按照左端點選擇，寫完後，範測也都對，很自信的送出，結果才22／100，蛤，難道是從右端點選，我換了另一種取法，結果0／100，這時拿出經典名句 ： 「proof by AC」,不過我猜測從左端點選法一定比右端點好，我來嘗試構造左端點取法的反例，過了許久，我構造出反例：是兩個區間重疊的時候，但我沒有想到要如何解決。

於是我先放下這題，去pC 撈完剩下的子題分，還算順利把要拿的分數都拿到了。
最後大概40分鐘，我有兩個選擇繼續pA還是pC，我心中大概預測pA應該多數人都有解出來，但我怎麼會卡那麼久，我到最後還是沒有想法。

最終以22／100／53／100（275／400）收場。

我個人認為沒有打的很好，結束後有聽其他人說我 pA 右側取法是好的，但我八成是哪裡寫錯了，好慘。我pA應該要寫出來的。
我預測很多人都會落在我的分數±50的區間。所以出來後就趕快開邪表。後來大概看了一下邪惡表格，落在rank 6，比預期差了一點，但也還行。

二模在 3/7 toi 初選，還有一個月。
加油！
